# DESIGN SYSTEM

## Overall Design Direction

The design should feel:

- modern
- trustworthy
- accessible premium
- calm
- spacious
- highly readable
- mobile-first
- organized

Reference aesthetics:
- Bilt
- Stripe
- Mercury
- Linear
- modern fintech UX

The experience should prioritize:
- clarity
- usability
- trust
- readability
- communication

NOT:
- luxury intimidation
- startup gimmicks
- visual noise
- excessive gradients
- excessive animation

---

# Typography

## Font Direction

Primary font:
- Geist
- Inter fallback

Typography should feel:
- clean
- modern
- neutral
- highly legible

---

## Headline Rules

Headlines should:
- be concise
- communicate quickly
- avoid buzzwords
- avoid oversized marketing copy

Preferred style:
- large
- bold
- readable
- calm

Avoid:
- giant paragraphs
- overly editorial typography
- luxury magazine feel

---

## Text Width Rules

Long text should NEVER span too wide.

Preferred content widths:
- paragraphs: max-w-2xl
- headlines: max-w-4xl

Readable line length is critical.

---

# Spacing System

Spacing is one of the most important aspects of the entire project.

The interface should feel:
- breathable
- intentional
- uncluttered

Preferred section spacing:
- desktop: py-24 to py-32
- mobile: py-16 to py-20

Card padding:
- p-6 to p-8

Avoid cramped layouts.

Whitespace creates trust.

---

# Container System

Primary container:
- max-w-[1280px]

Content container:
- max-w-[720px]

Centered layouts preferred.

---

# Border Radius

Preferred:
- rounded-2xl
- rounded-3xl

Avoid:
- sharp corporate corners
- overly pill-shaped UI

Buttons should NOT be fully pill-shaped.

---

# Shadows

Use:
- extremely subtle shadows
- soft borders
- restrained elevation

Avoid:
- heavy shadows
- floating glassmorphism cards
- visual noise

---

# Color Philosophy

The palette should feel:
- neutral
- calm
- modern
- professional

Primary background:
- off-white
- warm neutral

Primary text:
- near-black

Accent color:
- restrained
- trustworthy
- not overly saturated

Avoid:
- neon
- crypto gradients
- startup rainbow palettes

---

# Motion System

Animations should feel:
- subtle
- smooth
- modern
- restrained

Allowed:
- fade
- slight upward motion
- soft hover transitions

Avoid:
- bouncing
- flashy movement
- exaggerated parallax
- distracting motion

Motion should support clarity.

---

# Mobile UX Rules

Mobile experience is extremely important.

The site should:
- feel fast
- have large tap areas
- maintain readability
- preserve spacing
- avoid oversized text blocks

Sticky contact actions may be used:
- LINE
- WhatsApp

Mobile layouts should:
- stack cleanly
- avoid clutter
- prioritize CTA visibility

---

# Accessibility

Accessibility is a core design requirement.

Requirements:
- high contrast text
- readable font sizes
- semantic HTML
- keyboard accessibility
- accessible buttons
- proper spacing
- reduced motion support

Accessibility should feel integrated into the premium experience.

---

# Component Philosophy

Every component should:
- feel reusable
- feel modular
- feel intentional

Avoid:
- one-off inconsistent sections
- decorative complexity
- overdesigned cards

Consistency is critical.
